Overview

The okapi-lib distribution is a collection of Java classes design to help in programming components and applications related to translation and localization tasks. It offers a low-level resource model, as well as a set of common API to extract text parts from their original format, modify them, and merge them back. The library also provides facilities to manipulate, annotated and in general, work with the extracted text.

The distribution comes also with Tikal, a command-line tool to execute some simple localization-related tasks, like extraction and merge of XLIFF documents. If you want to use others Okapi tools (e.g. Rainbow), download one of the platform-specific okapi-apps distributions.

For a list of changes since the previous release, please see the Changes Log document.

Note that parts of this project are still relatively new and still in development. Some of the classes, interfaces and internal workings presented here may change in the upcoming versions of the Okapi Framework.

This distribution contains the following:

lib All the files needed to run the non-UI Okapi libraries. This directory includes the different dependencies some of the components need. See the JAR Dependencies section for details. If you want to use any of the Okapi UI classes, you need to download the okapi-apps distribution, which is platform-specific (because the UI classes use SWT).
help The help files that go with the several components provided with the library.
config Various default configuration files (e.g. segmentation rules, additional filter configurations, etc.)
examples A few Java examples on how to use the library. See the examples' readme file for details.
localweb/devguide The local copy of the Developer's Guide page for this release.
localweb/javadoc The local copy of the JavaDoc pages for this release.

The latest version of all documents and help is always available on-line, at the Okapi Framework Web site.

Requirements and Installation

Requirements

  • Windows: You must have a running installation of Java 1.5 or later.
  • Linux: You must have a running installation of Java 1.5 or later.
  • Macintosh: On both Mac OS X 10.5 (Leopard) or higher, and Mac OS X 10.4 (Tiger), the Okapi libraries requires Java 1.4.2 or Java 1.5.x.
    TODO->UPDATE THIS: Note that Java 1.6 (which is 64-bit) is not supported yet. Currently the Okapi libraries use Carbon, not Cocoa. Support for Cocoa will come when the Cocoa implementation of SWT is stable.

Installation

To install the Okapi libraries:

  1. Download the ZIP or the GZIP file of the distribution.
  2. Extract the file on any location.

Uninstallation

To uninstall the Okapi libraries

  1. To uninstall the distribution you can simply delete the folder where you have installed it.

Support

The examples directory contains source code of examples in Java that use the Okapi libraries.

If you have questions, suggestions or bug to report, you can do this different ways:

  • Read and post notes in the Okapi Tools users group (you need to be registered member to post, but not to read).
  • Read an post issue report on the Google Code project site.
