Borneo

Borneo is a simple Java application to manage localization projects.

Overview

A project is made of a set of documents that are extracted into a database. A document is any input that is supported by one of the filters provided along with Borneo. Currently there are filters for:

  • Java and .NET Properties files
  • JSON files

The extracted data go through different steps, the last one being the generation of the translated document. Borneo does not provide a translation environment, but creates translation packages for various translation tools.

Creating a New Project

1. Select the type of database to use

Borneo can currently work with two types of database: H2 and MySQL v5.x.

Borneo uses two locations to store a project: Extracted text, translations, language information, and most of the project data are stored in a database. But Borneo also keep track of the original files you are including to the project, and keeps multiple copies of them has they go through the localization process. These files are called "stored files" are are kept in a special directory structure.

  • Use H2 for local projects, where both the database and the stored files are located on local hard-drive. All files are in the same project directory. This server is embedded in Borneo and you have nothing to do to set it up.
  • Use MySQL for remote projects, where the database is located on a remote server, and the stored files are located on a shared network drive. Here database and stored files may be on different machines depending on your configuration. To use MySQL you must have MySQL installed on a server, and have access to database instance. MySQL is free and widely used.

When you start Borneo, no database type is selected. Once you have selected one, all project-related functions (open project, create new projects, etc.) are done in relation to the selected server. You can select a different serve at any time.

Choose the command Select Server from the File menu.

Select the database type. In this example, we will choose "Internal server - H2". And click OK.

You can see which server is selected at any time in the Settings tab.

2. Create the new project

Now that we have a server selected, we can create a new project. Note that if you do not have a server selected when creating a new project Borneo will prompt you to select one.

Select the command New Project from the File menu.

Enter the name of the project. The name must be a valid filename (so do not use characters forbidden in filenames).

Enter the project directory. The directory does not need to exist yet, Borneo will create all the path if necessary. Note that if you are using the MySQL server, instead of the project directory, you will be prompted for the directory were the stored files will reside. It must be a shared network directory.

3. Set the source root

Go to the Documents tab, and click the button with "..." just after the Source root display.

Enter the root directory of your source documents.

This is the common root for all the source documents that will go into the project. Choose it carefully, it determine the relative path for each source documents. It will be also used to calculate the location of the target documents in some cases.

Click OK when you have enter the root.

4. Define the project's settings

Go back to the Settings tab, click the Modify Settings button. This will allow you to change the source language for the project, as well as the default encoding for the source files. The source language must be a valid ISO language code, for example "EN-US". The encoding must be a valid encoding name such as "windows-1252" for example. Click Accept Changes to save your changes, or Discard Changes to cancel them.

You need also to add the target languages for the project. To do this, click the Add button in the Target language area.

In the Add Target Languages dialog box, select the languages you want to add in the Available languages list, and click on Add to move them to the Language to add list. Once you have selected all the languages you needed, click OK to add the languages to the project.

The next step is to set up the properties of each target language.

Select the target language for which you want to change the properties, and click Properties.

The Target Language Properties dialog box has several parameters that you can change:

  • Default encoding: It is the encoding used for the translated documents generated by Borneo (if no encoding is defined for a given target document).
  • Output root: This is an optional root to use instead of the source root. Leave the field empty to use the same root as the source document.
  • Default path mapping: These are the parameters that specify how the target document to named (if no path mapping is defined for a given target document). You can modify the filename's extension, replace some characters by others, add a prefix or a suffix, etc.

All the changes you make are reflected immediately at the bottom of the dialog box, where an example of the path of the source document is mapped with your parameters and displayed as it would be used for the target document.

Click OK when you have set you parameters.

Example of Process

1. Add the document to the project

Add a document to localize to the project by dropping it in the list of source documents in the Documents tab. You can also use the Add Documents command from the Documents menu.

2. Set the source document properties

Each document is associated to a filter and a default encoding. Borneo tries to guess these properties for you when you add the document to the project, but you may need to set them manually too. To do this: Select the documents in the Documents tab and use the Edit Source Document Properties command from the Documents menu.

The Source Document Properties dialog opens. Change the filter settings and default encoding as needed, and click OK.

3. Extract the translatable item into the source table

To extract the translatable text from the document into the source table: Select the Extract Source Documents command from the Source menu.

The Execute Action dialog box opens. This dialog box is used each time you launch an action.

Select what documents you want to process:

  • All of them
  • Only the documents that have been selected when you called this command.
  • Only the last selected document (its name is displayed at the bottom of the dialog)

Click Execute to start the action. Borneo calls the filters associated with each document and extract the text seen as translatable by the filter into the source table. The extracted items are flagged as Pending. You can list them in the Source tab by setting the check box labeled Show only pending items. Note that you may have already non-pending items for this document if you have done any extraction and updates before.

Note that if your input document is a bilingual file with existing translation (like a PO file), or if there are existing translated version of the source document (like for a Properties file), the existing translations is not lost. we will see how to take it into account later on.

4. Review the pending items

At this point you may want to review the pending items to see if the filter worked as expected. If you need to tune-up the parameters of the filter you can do this and simply re-extract the items again with the same command.

This step before the next one is important because the next step changes the status of the extracted items in the project and normally must be done only once for each new version of the source document, otherwise the status of the items may be incorrect.

Once you have verified the pending items, you can move to the next step.

5. Update the source table

To update the source table: Select the command Update Source Table from the Source menu.

Use the Execute Action dialog to specify the scope of the action and click Execute.

This action takes the pending items and compare them to any existing item already in the source table. The status of each item is updated according the changes brought up by the pending items (which are the latest version of the source document). Make sure to un-check the check box Show only pending items to see the normal source table.

6. Update the target table

The next step is to update the target table for each target language of the project: Select the command Update Target Table from the Target menu.

Use the Execute Action dialog to specify the scope of the action. This time, you can also select the scope of the target: either the one currently selected (see it in the toolbar) or all target languages. Click Execute to start the update.

This action copy the items of the source table into the target table, and update the status according the changes.

8. Export the translation package

If some items need translations or edit, you can export them into a translation package and send them away. The exported files can be merged back into the project at any time. If some update has occurred while the files were out for translation or edit, the status are updated accordingly.

To export a package: Select the command Export Translation Package from the Target menu.

Use the Execute Action dialog to specify the scope of the action. Click the Action Options button to set the export options.

The package is output in the XF sub-directory (eXchange Folder) in the Project directory.

9. Translate the document

The translation is done outside Borneo, with any translation tool that supports one of the translation package formats generated by Borneo.

10. Import the translation package

To import the exported packages back into the project you need to get the package back from the translator. He or she should not modify the structure or the names of the files, or touch the manifest file. Under these conditions, merging back a package (or some files in a package) is easily done:

Drag and drop the Manifest.xml file of the package you want to import in the main list of the Documents tab, or select the command Import Translation Package from the Target menu to select the Manifest.xml to use.

Once a manifest is selected, Borneo opens a dialog box where you can select which files of the package you want to import back.

12. Generate the target document

Note:

Before executing this action, make sure the path mapping for the target documents is set as you expect.

Once your documents are translated in the project, you want to generate the translated final files. Select the command Generate Target Documents from the Target menu.

Use the Execute Action dialog to specify the scope of the action and click Execute.

This action takes the translated items from the target table and generate a target file, based on the source document that was used last for the Update Target Table action. The document is output in the location and with the filename that you specify in the target properties for each target language.
