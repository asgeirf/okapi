Segmentation Rules Editor

This editor allows you to create, test and maintain segmentation rules stored in SRX documents.

Overview

SRX (Segmentation Rules eXchange) is a standard format to store segmentation rules. You can find the specifications of SRX 2.0 here: http://www.lisa.org/fileadmin/standards/srx20.html.

Segmentation rules are used to indicate to translation and other linguistic-oriented tools how to split a text into small meaningful sequence of words (for example in sentences).

This editor can load SRX version 1.0 documents, but they will be saved as version 2.0 documents. You will be warned both when opeing and saving SRX 1.0 documents. Be aware that versions 1.0 and 2.0 of SRX may be incompatible depending on how the version 1.0 document was interpreted by the tool that created the file. For more information about SRX versions, see the SRX Versions section.

Main Window

The main window is divided in two parts:

  • The upper part list all the rules for the group currently selected.
  • The lower part is the example area where you can try out the rules and see how they behave on a give text.

Language rules currently displayed -- Select the language group you want to work with. All the rules for the selected language group are display in the list below.

Groups and Options -- Click this button to open the Groups and Options dialog box where you can specify language groups and how they are mapped to languages.

Add -- Click this button to add a new rule at the end of the current group. This opens the Edit Rule dialog.

Edit -- Click this button to edit the rule currently selected. This opens the Edit Rule dialog.

Remove -- Click this button to remove the rule currently selected from the current group.

Move Up -- Click this button to move the rule currently selected upward in the group.

Move down -- Click this button to move the rule currently selected downward in the group.

Char Info -- Click this button to get information about the character currently selected in the Sample text edit box.

File Menu

New

Creates a new empty SRX document in the editor. If you have a modified un-saved document already opened, the tool will ask you if you want to save your changes before.

Shortcut: Ctrl+N

New With Simple Default Rules

Creates a new SRX document in the editor with simple default rules. Use this command as a quick way to start a new document, or to test a few rules. If you have a modified un-saved document already opened, the tool will ask you if you want to save your changes before.

Open

Opens an existing SRX document in the editor. If you have a modified un-saved document already opened, the tool will ask you if you want to save your changes before.

Shortcut: Ctrl+O

Save

Saves the current document. If the document has no name, the tool will prompt you to choose one. The tool will prompt you to confirm the save if the document was loaded as an SRX version 1.0 file. All documents are saved to SRX version 2.0,

Shortcut: Ctrl+S

Save As

Saves the current document under a new name.

Exit

Exits the editor. If you have a modified un-saved document already opened, the tool will ask you if you want to save your changes before.

Shortcut: Alt+F4

Tools Menu

Test Segmentation on a File

Takes an input file and segment it with the current SRX document and settings, and generate an output file with the segments marked up. See Test Segmentation on a File for more details on the requirements for the input document.
