Okapi Framework - Utilities

TTX Checker
Utilities Okapi Framework

During translation of Trados TTX files, target segmentation may in some cases prevent saving out the target after translation. The most common issue relates to inline codes. TagEditor has some requirements regarding the number and order of text segments and inline codes. This utility will identify potential problem areas.

This utility does not use filters.

Input Parameters

  • The list of the TTX documents to check (Input list 1).

Configuration

Options Tab

Check TTX

The only thing you need to do is add the documents to the input list. Target and Source language will be based on what's specified in each ttx file. Potential problem areas will be listed in the Rainbow Log (F9). Focus on the issues marked as "High" priority. In some cases you may need to address some "Low" priority issues as well. Open the ttx file TagEditor and address the issues in the Rainbow log.

For each issue the logfile will display two types of information. The first one is the logical pieces where a text segment is represented as a tag. Comparing source and target will gives you an idea where the problem is located. The second section shows the tags and the actual text. Use the text to locate the area in the file in TagEditor. The current way we're fixing issues is to add "dummy" text to empty segments. That dummy text can then be deleted once the target has been saved out.
