Utility – Byte-Order-Mark Conversion

Overview

This utility allows you to add a Byte-Order-Mark (BOM) to UTF-8 and UTF-16 files and remove the Byte-Order-Mark (BOM) from UTF-8, UTF-16, and UTF-32 files.

The Byte-Order-Mark (or BOM) is a special Unicode mark that is used to indicate if a file is in little-endian (U+FEFF) or big-endian (U+FFFE) mode. For more information on the BOM see http://www.unicode.org/faq/utf_bom.html.

This utility does not use filters.

Input Parameters

  • The list of the input documents to modify (Input list 1).
  • The names and locations of the output documents.

Configuration

Options Tab

Actions on the Byte-Order-Mark

Remove the Byte-Order-Mark if it is present -- Select this option to remove the BOM from the input files if one is detected. By default only the BOM of UTF-8 files are removed.

Remove also UTF-16 and UTF-32 BOMs -- Select this option to also remove the BOM from UTF-16 and UTF-32 files if one is detected. This is not something that is recommended: UTF-16 and UTF-32 files must have a BOM.

Add the Byte-Order-Mark if it is not already present -- Select this option to add a BOM in the input files if one is not detected. Note that the input files must be in UTF-8 or UTF-16. When adding a BOM this utility tries to determine if the encoding of the file is UTF-16. If it can't determine this it will assume the file is encoded as UTF-8. Because auto detection of UTF-16 file is not guaranteed to be 100% accurate, using UTF-8 files would be preferable. See the Encoding Conversion Utility for converting between encodings.
