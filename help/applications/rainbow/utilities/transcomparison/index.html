Okapi Framework - Utilities

Translation Comparison
Utilities Okapi Framework

This utility allows you to compare two sets of translated text.

The input is two documents, the first one containing the base translations, the second one containing the translations to compare to the base translations. The two documents must have the same number of entries and the entries must be in the same order.

For each entry the utility compares the two translated texts and calculates a score that measures how similar the text of the second document is to the text of the baseline translation. The score is a number between 0 and 100. The score is based on the Levenstein distance and additional processing. You can find more information on that metric at http://www.merriampark.com/ld.htm and http://en.wikipedia.org/wiki/Levenshtein_distance.

The utility outputs an HTML document for each pair of input documents. The HTML document lists the pairs of text and the resulting score. You can also generate a TMX document with the pairs and their scores.

Input Parameters

  • The list of the documents containing the base translation (Input list 1)
  • The list of the corresponding documents containing the translation to compare to the base one (Input list 2)
  • The source language.
  • The target language.
  • The default source encoding.
  • The default target encoding.

Configuration

Output Tab

HTML Output

Generate output tables in HTML -- Set this option to create one HTML file for each pair of input document, where the results of the comparisons are listed. The name of the HTML result file is the name of the document to compare with the base document, plus an .html extension.

Open the first HTML output after completion -- Set this option to open the first HTML result file created when the process is completed.

TMX Output

Generate a TMX output document -- Set this option to create one single TMX document that contains all the translations compared and their scores, for all input documents.

Enter the full path of the TMX document to generate. You can use the variable ${ProjDir} to specify the directory of the project.

Suffix for the second target language code -- Enter the suffix to append to the target language code for the second set of target entries. For example, enter -mt to indicate that the second target language is a machine-translation generated text. If you do not enter a suffix, the language code for both target entries will be the same and the TMX document will include only the last one.

Options Tab

Comparison Options

Ignore case differences -- Set this option to ignore case differences when calculating the scores.

Ignore white-space differences -- Set this option to ignore white-space differences when calculating the scores.

Ignore punctuation differences -- Set this option to ignore punctuation differences when calculating the scores.
