Segmentation Rules Editor

The Segmentation Rules Editor allows you to create, test and maintain sets of SRX 2.0 documents.

Overview

SRX (Segmentation Rules eXchange) is a standard format to store segmentation rules. You can find the specifications of SRX 2.0 here: http://www.lisa.org/fileadmin/standards/srx20.html.

The editor can load SRX version 1.0 documents, but they will be saved as version 2.0 documents. Be aware that versions 1.0 and 2.0 of SRX may be incompatible depending on how the version 1.0 document was interpreted by the tool that created the file.

Main Window

The main window is divided in two parts:

  • The upper part list all the rules for the group currently selected.
  • The lower part is the example area where you can try out the rules and see how they behave on a give text.

Language rules currently displayed -- Select the language group you want to work with. All the rules for the selected language group are display in the list below.

Groups and Options -- Click this button to open the Groups and Options dialog box where you can specify language groups and how they are mapped to languages.

File Menu

New

Creates a new empty SRX document in the editor. If you have a modified un-saved document already opened, the tool will ask you if you want to save your changes before.

Shortcut: Ctrl+N

Open

Opens an existing SRX document in the editor. If you have a modified un-saved document already opened, the tool will ask you if you want to save your changes before.

Shortcut: Ctrl+O

Save

Saves the current document. If the document has no name, the tool will prompt you to choose one. The tool will prompt you to confirm the save if the document was loaded as an SRX version 1.0 file. All documents are saved to SRX version 2.0,

Shortcut: Ctrl+S

Save As

Saves the current document under a new name.

Exit

Exits the editor. If you have a modified un-saved document already opened, the tool will ask you if you want to save your changes before.

Shortcut: Alt+F4
