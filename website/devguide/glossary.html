Code type

The code type of a Code object is an indicator of what this code represent. Several code types are pre-defined and should be used whenever possible. for example: Code.TYPE_BOLD, Code.TYPE_ITALIC, Code.TYPE_UNDERLINED, Code.TYPE_IMAGE, Code.TYPE_LINK, etc.

The code type of a code is used extensively to link opening and closing codes: both opening and closing codes of the same pair must have the same code type. You can use null for code type, but then loose any pairing mechanism.

As opposed to the tag type, a code type is never changed automatically by the basic methods of the text unit objects.

Event

An event is object used to carry information through the pipeline. Events can be associated with resources, and in the case of filter events, with skeleton.

  • The START and FINISHED events are always the first and last events going through the pipeline.
  • The FILE_RESOURCE event is used to pass along a complete document.
  • The filter events (START_DOCUMENT, START_SUBDOCUMENT, START_GROUP, DOCUMENT_PART, TEXT_UNIT, END_GROUP, END_SUBDOCUMENT, and END_DOCUMENT) are used to pass along a document broken down into parts where translatable and non-translatable content is stored.

Filter writer

A filter writer class is an implementation of the IFilterWriter interface. Its purpose is to output the document processed by a filter. The format of the output can be different for each filter writer. There is normally one filter writer that is capable of re-constructing the original format of the input document.

Inline code marker

An inline code marker is a pair of two special Unicode characters that are inside a coded text to hold the place of an inline code.

The marker is composed of the following two special characters:

  • A code marker that indicates if the inline code should be represented as an opening (TextFragment.MARKER_OPENING), closing (TextFragment.MARKER_CLOSING), or isolated (TextFragment.MARKER_ISOLATED) code, or if it is a segment (TextFragment.MARKER_SEGMENT). These values are U+E101, U+E102, U+E103, and U+E104 (part of the Private Use Area of Unicode).
  • An index value that points to a Code object in the list of codes for the TextFragment where this code occurs. The value is the zero-based index of the code + 57616 converted to a character: that is 0 is U+E110, 1 is U+E111, 2 is U+E112, etc. Those characters are in the Private Use Area of Unicode and allow for several thousands of index values.

In the example below, the inline code markers for "<b>" and "</b>" are highlighted. (Note for display purpose, the special characters are noted here as \uHHHH where HHHH is there Unicode value, but they are raw characters in the coded text string.

Normal text: "Text in <b>bold</b>"
 Coded text: "Text in \uE101\u110bold\uE102\uE111"

        |     +--- Index of the code

Text unit

The text unit is the basic item used by a filter to store extracted text and its associated information, it is implemented by the TextUnit class. Text unit resources are associated with the TEXT_UNIT events generated by the filters.
